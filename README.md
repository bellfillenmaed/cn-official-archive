# cn-official-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存储，旨在提供一种便捷的访问与分享方式，不针对任何单个域名或具体网站。

## 项目简介

cn-official-archive 是一个基于 GitHub Pages 的静态页面集合。仓库内包含多个独立的 HTML 文件，每个文件对应一个独立的归档内容。用户可以通过浏览仓库目录或访问对应的 GitHub Pages 链接来查看这些页面。

该项目仅作为信息归档用途，不涉及任何动态交互或后端服务。

## 目录说明

```
├── pages/           # 存放所有归档的 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/          # 页面所需的静态资源（CSS、图片等，如有）
├── index.html       # 仓库首页（可选）
└── README.md        # 本文件
```

- `pages/`：核心目录，每个 HTML 文件代表一个独立的归档页面。
- `assets/`：存放页面引用的样式、图片等资源文件。
- `index.html`：可选的仓库首页，用于展示页面列表或导航。

## 页面归档说明

每个 HTML 页面均为独立的归档文件，内容可能包括文本、图片、链接等静态元素。页面以原始形式保存，未进行动态生成或外部数据加载。

### 如何添加新页面

1. 将 HTML 文件放入 `pages/` 目录。
2. 确保页面内所有资源（如图片、样式）使用相对路径或已上传至 `assets/` 目录。
3. 提交 Pull Request 或直接推送到主分支。

### 如何查看已归档页面

- 直接访问仓库的 GitHub Pages 链接（需启用 Pages 功能）。
- 或在仓库文件列表中找到对应 HTML 文件，点击后可通过浏览器查看渲染效果。
